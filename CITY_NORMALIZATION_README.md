# Нормализация городов - Инструкция по развертыванию

## Описание изменений

Добавлена система нормализации названий городов для улучшения поиска анкет по городу. Теперь пользователи с разными вариантами написания одного города (МСК, Москва, Moscow и т.д.) будут находить друг друга.

## Что изменилось

### Новые файлы
- `bot/utils/city_normalizer.py` - модуль нормализации городов
- `alembic/versions/001_add_normalized_city.py` - миграция базы данных
- `migrate_existing_cities.py` - скрипт для нормализации существующих данных
- `tests/test_city_normalizer.py` - тесты нормализации

### Измененные файлы
- `bot/database/models.py` - добавлено поле `normalized_city` с индексом
- `bot/database/requests.py` - нормализация при сохранении и поиск по нормализованному полю

## Порядок развертывания

### 1. Применение миграции базы данных

Сначала примените миграцию для добавления нового поля `normalized_city`:

```bash
# Если используете Docker
docker-compose exec db psql -U postgres -d dota_tinder -f alembic/versions/001_add_normalized_city.py

# Или через Alembic (если настроен)
alembic upgrade head
```

### 2. Нормализация существующих данных

После применения миграции запустите скрипт для нормализации городов существующих пользователей:

```bash
# Если используете Docker
docker-compose exec bot python migrate_existing_cities.py

# Локально
python migrate_existing_cities.py
```

Скрипт:
- Прочитает всех пользователей из базы данных
- Нормализует их города с помощью словаря алиасов
- Обновит поле `normalized_city`
- Выведет статистику по обновлениям

### 3. Перезапуск бота

После миграции и нормализации данных перезапустите бота:

```bash
docker-compose restart bot
```

## Поддерживаемые алиасы городов

### Москва
- мск, msk, moscow, moskow → москва

### Санкт-Петербург
- спб, spb, питер, петербург, saint petersburg → санкт-петербург

### Екатеринбург
- екб, ekb, yekaterinburg → екатеринбург

## Добавление новых городов

Для добавления нового города в словарь алиасов отредактируйте `bot/utils/city_normalizer.py`:

```python
CITY_ALIASES: Dict[str, str] = {
    # Существующие алиасы...
    
    # Ваш новый город
    "новосибирск": "новосибирск",
    "нск": "новосибирск",
    "nsk": "новосибирск",
}
```

После изменения словаря перезапуск бота не требуется - изменения применятся мгновенно.

## Как это работает

1. **При регистрации/редактировании профиля:**
   - Пользователь вводит город (например, "МСК")
   - Система нормализует его ("москва")
   - В поле `city` сохраняется оригинал ("МСК") для отображения
   - В поле `normalized_city` сохраняется нормализованное значение ("москва")

2. **При поиске анкет:**
   - Система сравнивает `normalized_city` пользователей
   - Пользователи с "МСК", "Москва", "Moscow" будут иметь одинаковый `normalized_city` ("москва")
   - Они будут находить друг друга при поиске по городу

3. **Для неизвестных городов:**
   - Если город не в словаре алиасов, он просто очищается (lowercase, trim, удаление спецсимволов)
   - Например, "Тула" → "тула"

## Откат изменений

Если необходимо откатить изменения:

```sql
-- Удалить индекс
DROP INDEX IF EXISTS ix_users_normalized_city;

-- Удалить колонку
ALTER TABLE users DROP COLUMN IF EXISTS normalized_city;
```

Затем удалите новые файлы и восстановите старые версии измененных файлов из git.
